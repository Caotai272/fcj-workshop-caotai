**Trong trang này** bạn sẽ cần giới thiệu worklog của bạn **như thế nào**? Bạn hoàn thành chương trình trong vòng **bao nhiêu tuần**? Bạn **đã làm gì** trong các tuần đó?


Thông thường và cũng là tiêu chuẩn, một worklog được thực hiện trong khoảng 3 tháng (trong suốt thời gian thực tập) với nội dung các tuần như sau:

**Tuần 1:** [Làm quen với AWS, Cloud Computing, Region, Availability Zone và AWS Management Console.](1.1-week1/)

**Tuần 2:** [Tìm hiểu IAM, quản lý User, Group, Role, Policy và phân quyền truy cập trên AWS.](1.2-week2/)

**Tuần 3:** [Tìm hiểu VPC, Subnet, Route Table, Internet Gateway, NAT Gateway và Security Group.](1.3-week3/)

**Tuần 4:** [Tìm hiểu EC2, Key Pair, Elastic IP, Load Balancer và Auto Scaling.](1.4-week4/)

**Tuần 5:** [Tìm hiểu HCL AppScan, Nessus và Burp Suite để kiểm thử bảo mật ứng dụng và hệ thống.](1.5-week5/)

**Tuần 6:** [Tìm hiểu Amazon S3, EBS, EFS và các hình thức lưu trữ dữ liệu trên AWS.](1.6-week6/)

**Tuần 7:** [Tìm hiểu Amazon RDS, DynamoDB, backup, restore và quản lý cơ sở dữ liệu trên AWS.](1.7-week7/)

**Tuần 8:** [Tìm hiểu Migration lên AWS với VM Import/Export, AWS DMS và Schema Conversion Tool.](1.8-week8/)

**Tuần 9:** [Tìm hiểu CloudWatch, CloudTrail, log, metric, alarm và giám sát hệ thống.](1.9-week9/)

**Tuần 10:** [Tìm hiểu AWS Budgets, Cost Explorer, KMS, WAF và tối ưu chi phí, bảo mật hệ thống.](1.10-week10/)

**Tuần 11:** [Tìm hiểu Lambda, API Gateway, Cognito, SQS, SNS và SES trong mô hình serverless.](1.11-week11/)

**Tuần 12:** [Tổng hợp kiến thức, triển khai demo trên AWS, kiểm thử, troubleshooting và hoàn thiện báo cáo.](1.12-week12/)
