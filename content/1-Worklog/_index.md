**On this page**, you will need to introduce your worklog. **How** did you complete it? How many weeks did you take to complete the program? **What** did you do in those weeks?

Typically, and as a standard, a worklog is carried out over about 3 months (throughout the internship period) with weekly contents as follows:

**Week 1:** [Getting familiar with AWS, Cloud Computing, Region, Availability Zone and AWS Management Console.](1.1-week1/)

**Week 2:** [Exploring IAM, managing User, Group, Role, Policy and access permissions on AWS.](1.2-week2/)

**Week 3:** [Exploring VPC, Subnet, Route Table, Internet Gateway, NAT Gateway and Security Group.](1.3-week3/)

**Week 4:** [Exploring EC2, Key Pair, Elastic IP, Load Balancer and Auto Scaling.](1.4-week4/)

**Week 5:** [Exploring HCL AppScan, Nessus and Burp Suite to test application and system security.](1.5-week5/)

**Week 6:** [Exploring Amazon S3, EBS, EFS and data storage options on AWS.](1.6-week6/)

**Week 7:** [Exploring Amazon RDS, DynamoDB, backup, restore and database management on AWS.](1.7-week7/)

**Week 8:** [Exploring Migration to AWS with VM Import/Export, AWS DMS and Schema Conversion Tool.](1.8-week8/)

**Week 9:** [Exploring CloudWatch, CloudTrail, log, metric, alarm and system monitoring.](1.9-week9/)

**Week 10:** [Exploring AWS Budgets, Cost Explorer, KMS, WAF and cost optimization, system security.](1.10-week10/)

**Week 11:** [Exploring Lambda, API Gateway, Cognito, SQS, SNS and SES in the serverless model.](1.11-week11/)

**Week 12:** [Applying knowledge to the project, deploying a demo on AWS, testing, troubleshooting and finalizing the report.](1.12-week12/)
